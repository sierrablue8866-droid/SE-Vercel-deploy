Setup Instructions

Before running the seed, you need to:

  1. Create a Supabase project at supabase.com (free tier works)
  2. Open SQL Editor → paste the contents of schema.sql → click Run
  3. This creates all tables (compounds, listings, inquiries, etc.) + RLS policies
  4. Go to Project Settings → API → copy Project URL and anon public key
  5. Edit supabase-config.js and paste your values
  6. Set window.SIERRA_SUPABASE_ENABLED = true in that file
  7. Refresh this page — the status above should turn green

⚠️ Important: After seeding, create your admin user. Sign up via Supabase Auth, then run this in SQL Editor:

INSERT INTO users (id, email, name, role)
VALUES ('YOUR-AUTH-UID', 'your@email.com', 'Ahmed', 'admin');

To get your AUTH UID: Go to Authentication → Users → copy the UID column.

After Seeding

  • The SE portal will automatically use Supabase (via supabase.js)
  • If Supabase is unavailable, it falls back to data.js (always works)
  • Inquiry form will write to inquiries table (with localStorage fallback)
  • Career form will write to career_applications table
  • Real-time: compound/listing changes in admin reflect instantly on the site
  • To enable Supabase on the live site, the SDK is already loaded in index.html
